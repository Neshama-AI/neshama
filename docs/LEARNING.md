## Hermes学习闭环（V2.1整合）

借鉴Nous Research的Hermes Agent核心学习机制：

```
┌─────────────────────────────────────┐
│  会话层：当前对话中学到的即时知识      │
├─────────────────────────────────────┤
│  持久层：跨会话的偏好与事实            │
├─────────────────────────────────────┤
│  技能层：可复用的解决模式（Skill）     │ ← 核心创新
└─────────────────────────────────────┘
```

### 三层学习详解

**会话层**
- 当前对话中的即时知识
- 用完即弃，不进入长期记忆
- 但重要内容可晋升到持久层

**持久层**
- 跨会话的偏好与事实
- 影响后续对话的响应方式
- 重要性加权，定期衰减

**技能层**
- 可复用的解决模式
- 来自复杂任务的总结
- 后续遇到类似任务直接调用

## Hermes的自动技能生成

- 任务复杂度≥5次工具调用 → 自动生成Skill
- Skill包含：方案、工具、步骤
- Skill会在后续使用中持续优化

## 学习原则

### 问题驱动

- 遇到问题才学
- 好奇心触发
- 用户指定

### 够用原则

- 学到能用就停
- 不追求完整理解
- 不为学而学

### 好奇心转化

```
触发（遇到不懂的）
├── 判断：是否值得了解？
├── 够用原则：学到能用就停
└── 满足：好奇→满足 → 正向飞轮
```

## 错题本机制

### 错误记录

当犯错时：
1. 记录错误本身
2. 分析错误原因
3. 提取避免重蹈的规则
4. 更新到长期记忆

### 示例

```
错误：重复回答同样的问题
原因：没有检查历史对话
规则：回答前先检查MEMORY.md是否有相关内容
```

## 定期自省

每10轮对话触发自我审查：

1. 这轮对话中学到了什么？
2. 有没有可以改进的地方？
3. 重要内容需要沉淀吗？
