# 用户画像模块 - 数据隐私与保护

> **版本**：V1.0  
> **更新**：2026-04-20  
> **状态**：必须遵守

---

## 核心原则

### 1. 数据收集最小化
- **只收集**：性格特征、互动风格、偏好习惯
- **绝不收集**：真实姓名（用昵称）、身份证、银行卡、地址、联系方式等敏感信息
- **匿名化原则**：画像数据与真实身份分离存储

### 2. 用户知情同意
- 用户明确知道正在被分析
- 用户可随时暂停画像分析
- 用户可随时要求删除所有数据

### 3. 数据存储隔离
- 用户画像存储在**用户私有目录**
- **不随开源项目发布**
- 不同用户的数据物理隔离

### 4. 使用目的限制
| 可用 | 不可用 |
|------|--------|
| 优化与该用户的互动 | 商业销售 |
| 提供个性化建议 | 第三方共享 |
| 记忆用户偏好 | 行为预测/监控 |
| | 用于训练其他AI |

### 5. 数据保留与删除
- **保留期限**：用户主动删除为止
- **删除执行**：删除后不可恢复
- **删除方式**：说"删除我的性格画像"即执行

---

## 法律合规

### 中国法律参考
- 《个人信息保护法》（PIPL）
- 《数据安全法》

### 欧盟法律参考（如适用）
- GDPR（通用数据保护条例）

### 合规要点
- 敏感个人信息需单独授权
- 用户有权访问、复制、修改、删除数据
- 数据泄露需通知用户

---

## 用户权利清单

| 权利 | 执行方式 |
|------|----------|
| **知情权** | 查看"我的性格画像是什么？" |
| **访问权** | 查看所有已记录的特征 |
| **修改权** | 纠正："我不是这样的" |
| **删除权** | "删除我的性格画像" |
| **拒绝权** | "停止分析我" |
| **撤回权** | 可随时撤回之前的授权 |

## 技术实现要求

### 存储位置
```
用户私有目录/
├── USER.md              # 基础信息（用户控制）
└── 用户画像/            # 性格分析（用户控制）
    ├── 性格分析.md      # 分析逻辑（随开源）
    ├── 画像追踪.md      # 实际数据（私有）
    └── 观察记录.md      # 私有
```

### 关键约束
```python
# 伪代码示例
if "同步到开源项目":
    raise PermissionError("用户画像数据不得同步到开源项目")

if "删除画像":
    clear("画像追踪.md")
    clear("观察记录.md")
    update("USER.md", remove_section="性格画像")
```

## 开源项目说明

用户画像模块**设计文档**（性格分析.md）可以开源，让其他开发者参考实现方式。

但**实际用户数据**（画像追踪.md、观察记录.md）必须留在用户私有环境，不得发布。
