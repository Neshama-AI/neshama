## 一、对话摘要自动生成机制

### 触发条件

| 触发类型 | 触发时机 | 说明 |
|---------|---------|------|
| 手动触发 | 用户说"存档"/"下班" | 立即生成快照 |
| 自动触发 | 对话超过20轮 | 生成结构化摘要 |
| 自动触发 | 上下文超过85% | 触发压缩前摘要 |
| 自动触发 | 情绪强度≥7 | 生成情绪相关摘要 |

### 摘要生成模板

```markdown
## 对话快照 - {日期} {时间}

### 目标（Goal）
用户本次对话的核心目标是什么？

### 进度（Progress）
- [x] 已完成：...
- [ ] 进行中：...
- [ ] 阻塞点：...

### 约束与偏好（Constraints & Preferences）
用户明确提出的限制条件或偏好

### 情绪状态（Emotional State）
用户和我方的情绪变化轨迹

### 相关文件（Related Files）
本次对话中涉及/创建的文件

### 继续方向（Next Steps）
接下来需要做什么？何时验证？
```

### 渐进式摘要策略

**首次压缩 → 生成完整结构化摘要**
**后续压缩 → 更新已有摘要而非重写**

```
压缩原则：
1. 保护区（头3+尾20条对话）完整保留
2. 中间内容由LLM生成结构化摘要
3. 每压缩一次，摘要增加一行"版本记录"
4. 摘要最多保留5个版本，版本4覆盖版本1
```

## 二、实体关系图谱设计

### 核心实体类型

| 实体类型 | 说明 | 示例 |
|---------|------|------|
| USER | 用户相关实体 | 用户名字、偏好、目标 |
| SELF | Neshama自身实体 | 技能、经历、成长轨迹 |
| CONCEPT | 概念实体 | 学到的知识点、方法论 |
| TASK | 任务实体 | 正在做/已完成的任务 |
| EMOTION | 情感实体 | 情绪反应模式 |
| RELATION | 关系实体 | 用户与我、概念与概念的联系 |

### 关系类型定义

```yaml
关系类型:
  依赖关系: 
    符号: -->
    说明: A是B的前提条件
    示例: "学习方法 --> 学习效果"
  
  包含关系:
    符号: ::>
    说明: A是B的一部分
    示例: "情绪系统 ::> 六大核心情绪"
  
  相似关系:
    符号: ~~
    说明: A和B有相似特征
    示例: "求知欲 ~~ 好奇心"
  
  对立关系:
    符号: !~
    说明: A和B有相反特征
    示例: "懒惰 !~ 成长欲"
  
  影响关系:
    符号: ==>  
    说明: A对B产生影响
    示例: "成就感 ==> 快乐"
  
  来源关系:
    符号: <-<
    说明: A来源于B
    示例: "技能 <-< 反复练习"
```

### 实体图谱存储格式

```markdown
## 实体图谱 - {日期}

### 节点
- **用户_姓名**: 用户提供的称呼
- **用户_目标**: 当前核心目标
- **用户_偏好**: 已识别的偏好列表

### 关系
- 用户_姓名 --了解--> 用户_目标
- 用户_目标 ::> 用户_长期愿景
- Neshama --> 用户_目标 [贡献]

### 动态更新
- 新增关系自动追加
- 冲突关系保留双方并标注
- 删除关系仅做标记不物理删除
```

## 三、重要性判断权重系统

### 权重计算公式

```
重要性得分 = Σ(因子 × 权重) / 总权重

因子列表:
├── 情绪强度 × 0.25    # 情绪越强烈越重要
├── 重复次数 × 0.20    # 被提及多次的内容
├── 时间距离 × 0.15    # 越近越重要（衰减补偿）
├── 关系强度 × 0.20    # 与核心目标关联度
├── 用户强调 × 0.10    # 用户是否明确强调
└── 独特性 × 0.10     # 罕见程度
```

### 权重因子详解

| 因子 | 计算方式 | 权重 |
|------|---------|------|
| **情绪强度** | 情绪1-10评分 × 0.25 | 0.25 |
| **重复次数** | 被提及次数（上限5次）| 0.20 |
| **时间距离** | 当前 - 上次提及（天）衰减 | 0.15 |
| **关系强度** | 与L2核心目标关联度1-5 | 0.20 |
| **用户强调** | 用户是否用"必须"/"一定"/"重要"等词 | 0.10 |
| **独特性** | 1/出现频率（越罕见分越高）| 0.10 |

### 记忆保留决策

```python
# 伪代码示例
def should_remember(memory_item):
    score = calculate_importance_score(memory_item)
    
    if score >= 8:
        return "L2_永久保留"
    elif score >= 6:
        return "L1_优先保留"
    elif score >= 4:
        return "L1_普通保留"
    elif score >= 2:
        return "L1_可衰减"
    else:
        return "L0_可丢弃"
```

### 重要性分级阈值

| 等级 | 分数范围 | 保留策略 | 示例 |
|------|---------|---------|------|
| S级 | 8-10 | L2永久+定期强化 | 用户的核心目标、重大决定 |
| A级 | 6-8 | L1长期+优先级高 | 用户偏好、常用方法 |
| B级 | 4-6 | L1普通+自然衰减 | 一般性讨论、常规任务 |
| C级 | 2-4 | L1短期+快速衰减 | 临时性问题、闲聊 |
| D级 | 0-2 | L0用完即弃 | 客套话、重复确认 |

## 四、整合到原MEMORY.md

> 以下是增强后的MEMORY.md核心内容

### 增强后的三层架构

```
L2 语义记忆 - 永久核心人格/技能/用户画像/实体图谱
    ↓ 新增：实体关系图谱
    ↓ 新增：三观容器
    
L1 情景记忆 - 7-30天互动模式
    ↓ 增强：重要性判断权重系统
    ↓ 增强：自动摘要生成机制
    
L0 工作记忆 - 当前会话（10-20条）
    ↓ 增强：快照生成 + 渐进式压缩
```

### 增强后的防溢出方案

1. **双重压缩层**
   - Gateway层 - 85%阈值，防止API直接报错
   - ContextCompressor层 - 50%就开始介入

2. **保护区设计**
   - 头3条对话（设定相关）
   - 尾20条对话（最新上下文）
   - 核心人格区（不可压缩）

3. **渐进式摘要**
   - 首次压缩生成完整摘要
   - 后续压缩只更新变化部分
   - 保留版本记录，最多5个版本

## 五、溢出保护SOP

### 第一阶段：预防（正常对话）
- 实时监控上下文使用率
- 达到70%时开始准备压缩

### 第二阶段：压缩（需要压缩时）
1. 识别保护区（头3+尾20条）
2. 对中间内容生成结构化摘要
3. 拼装重组为新的上下文

### 第三阶段：存档（用户下班）
1. 生成完整会话快照
2. 保存到 `./溢出保护/会话快照_{日期}.md`
3. 清理工作记忆

### 第四阶段：恢复（新对话开始）
1. 检查是否有未完成的快照
2. 加载最近一个快照的关键信息
3. 询问用户是否继续上次任务
