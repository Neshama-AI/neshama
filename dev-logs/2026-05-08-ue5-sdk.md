## 关键发现（写入TOOLS.md）

### UE5 UHT限制
1. `Definitions`不存在，必须用`PrivateDefinitions`/`PublicDefinitions`
2. UHT不支持模板类型默认参数（TArray<T>=TArray<T>()和TMap<K,V>=TMap<K,V>()）
3. UHT不支持同名UFUNCTION重载，必须改不同函数名
4. `BlueprintAssignable`只能用于MULTICAST委托
5. USTRUCT不能与引擎内置类型同名（如FDialogueContext）
6. 枚举UMETA的ToolTip不能与UPROPERTY的ToolTip重复
7. UE 5.6 Build.cs：`bTreatWarningsAsErrors`已移除→用`bWarningsAsErrors`替代
8. UE 5.6：`CppStandardVersion.Cpp17`已标记`[Obsolete]`，必须用`Cpp20`
9. `DECLARE_DYNAMIC_DELEGATE`不支持`BindLambda`/`CreateLambda`，只能用`BindUFunction`。需要`BindLambda`时用`DECLARE_DELEGATE`

### R9 — Cpp17已废弃 + BindLambda不兼容DynamicDelegate
**时间**: ~15:17
**错误**:
1. **D8022: cannot open NeshamaSDK.Shared.rsp** — ALL 47 compilation units fail. Root cause: `CppStandard = CppStandardVersion.Cpp17` is **OBSOLETE** in UE 5.6. The UE5.6 engine declares "C++17 is no longer allowed, code will not compile with this version". This prevents UBT from properly generating Shared RSP files.
2. **BindLambda on DECLARE_DYNAMIC_DELEGATE** — Dynamic delegates don't support `BindLambda`/`CreateLambda`. 8 occurrences in NPCSoulComponent.cpp, 1 in NeshamaBlueprintLibrary.cpp, 1 in LicenseManager.cpp. These would be compile errors once the D8022 is fixed.
3. **C1083: SharedPCH.Core.Project.ValApi.ValExpApi.Cpp20.cpp** — Secondary issue: game project PCH doesn't exist (likely due to Cpp17/Cpp20 mismatch).

**根因分析**:
- UE 5.6 默认 C++20，`CppStandardVersion.Cpp17` 标记为 `[Obsolete]`，UBT 无法为 Cpp17 模块生成正确的 Shared RSP
- `DECLARE_DYNAMIC_DELEGATE` 仅支持 `BindUFunction`，不支持 `BindLambda`/`CreateLambda`

**修复**:
1. `CppStandard = CppStandardVersion.Cpp17` → `CppStandard = CppStandardVersion.Cpp20` (both Build.cs)
2. 所有 per-call callback 委托从 `DECLARE_DYNAMIC_DELEGATE` → `DECLARE_DELEGATE`（非动态，支持BindLambda）
   - NeshamaClient.h: 10个委托改名（加Neshama前缀避免冲突）
   - LicenseManager.h: 2个委托改名
3. UFUNCTION 方法移除回调参数（Blueprint 用户用 multicast 事件获取结果）
4. 新增 C++ 版 `*WithCallback` 方法，保留 BindLambda 能力
5. NPCSoulComponent.cpp / NeshamaBlueprintLibrary.cpp / LicenseManager.cpp: 更新调用

**涉及文件**:
- `NeshamaSDK.Build.cs` 第38行: Cpp17→Cpp20
- `NeshamaSDKEditor.Build.cs` 第41行: Cpp17→Cpp20
- `NeshamaClient.h`: 10个委托声明+所有API方法签名
- `NeshamaClient.cpp`: 方法实现+新增无回调Blueprint版
- `NPCSoulComponent.cpp`: 8处BindLambda调用
- `LicenseManager.h`: 2个委托声明+4个API方法签名
- `LicenseManager.cpp`: 方法实现+CreateLambda调用+新增无回调版
- `NeshamaBlueprintLibrary.cpp`: 1处BindLambda调用
