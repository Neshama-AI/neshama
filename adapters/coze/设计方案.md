# 扣子适配器设计方案

> 版本：V1.0
> 日期：2026-04-20
> 目标：将Neshama灵魂配方转换为扣子平台可导入的配置文件

---

## 一、核心问题解答

### 1.1 扣子Agent的配置结构

扣子Bot的核心配置包括：

| 配置模块 | 说明 | Neshama对应 |
|---------|------|-------------|
| **基本信息** | 名称、描述、头像、欢迎语 | NAME.md |
| **人设Prompt** | 最核心的配置，决定Bot行为 | SOUL.md |
| **模型参数** | temperature、max_tokens等 | 需额外配置 |
| **技能** | 插件、知识库 | Skills/*.md |
| **记忆** | 长期记忆、变量 | MEMORY.md |
| **工作流** | 复杂逻辑编排 | 自动化脚本 |

### 1.2 扣子是否支持导出/导入Agent配置

**关键发现：**
- ✅ 工作流支持导出为ZIP格式（加密）
- ✅ 个人进阶版及以上支持导入导出
- ❌ **Bot整体配置无法直接导出**（无官方API）
- ⚠️ 通过API可查看Bot配置，但无法直接复制

**替代方案：**
1. **Prompt复制**：最核心的"人设与回复逻辑"可直接复制粘贴
2. **手动配置辅助**：提供详细的配置清单，用户手动填写
3. **配置文件格式**：生成结构化配置文件，用户照填

### 1.3 Neshama配方 → 扣子配置的映射关系

```
Neshama灵魂配方                    扣子配置
┌─────────────────────┐           ┌─────────────────────┐
│  SOUL.md            │ ──────→  │  人设与回复逻辑      │
│  (人格灵魂)          │           │  (System Prompt)     │
├─────────────────────┤           ├─────────────────────┤
│  MEMORY.md          │ ──────→  │  长期记忆配置        │
│  (记忆系统)          │           │  + 变量设计          │
├─────────────────────┤           ├─────────────────────┤
│  Skills/*.md        │ ──────→  │  技能(插件/知识库)   │
│  (技能包)            │           │                     │
├─────────────────────┤           ├─────────────────────┤
│  情绪/欲望系统       │ ──────→  │  在Prompt中模拟实现  │
│  (高级特性)          │           │                     │
├─────────────────────┤           ├─────────────────────┤
│  说话风格指南        │ ──────→  │  回复策略/语气设置  │
├─────────────────────┤           ├─────────────────────┤
│  边界系统           │ ──────→  │  约束条件           │
└─────────────────────┘           └─────────────────────┘
```

## 二、适配器输出格式

### 2.1 核心输出：人设Prompt模板

**文件结构：**
```
扣子适配器/
├── prompt_template.md      # 核心人设Prompt（直接复制）
├── config_checklist.md      # 配置清单（手动填写）
├── skills_mapping.md        # 技能映射说明
└── 导入指南.md              # 一步步导入教程
```

### 2.3 配置清单（手动填写项）

```markdown

> Neshama生成的灵魂配方需要配合以下手动配置才能完整生效

## 基本信息

| 配置项 | 填写内容 | 说明 |
|-------|---------|------|

| 欢迎语 | __________ | 首次对话的开场白 |

| 参数 | 推荐值 | 适用场景 |
|-----|-------|---------|
| Temperature | 0.7-0.8 | 平衡模式，兼顾准确与创造 |
| Max Tokens | 2000-4000 | 根据回复长度需求调整 |

## 技能配置建议

### 建议启用的插件
- [ ] 联网搜索（如需要实时信息）
- [ ] 图片理解（如需要分析图片）
- [ ] 代码执行（如需要运行代码）

### 知识库建议
- [ ] 上传与专业领域相关的文档
- [ ] 定期更新知识库内容

## 记忆配置

### 长期记忆
- [ ] 开启长期记忆功能
- [ ] 配置记忆保留时长（建议30天）

### 用户变量
- [ ] 配置用户偏好变量
- [ ] 配置对话上下文变量

## 发布设置

- [ ] 配置发布渠道（豆包/飞书/微信等）
- [ ] 设置访问权限
```

## 三、导入操作指南

### 3.1 第一步：创建Bot

1. 登录 [扣子平台](https://www.coze.cn/)
2. 进入工作空间
3. 点击「创建智能体」
4. 填写基本信息（名称、描述）

### 3.2 第二步：配置人设

1. 进入Bot编辑页面
2. 找到「人设与回复逻辑」输入框
3. 复制 `prompt_template.md` 中的全部内容
4. 粘贴到输入框中
5. 根据实际情况修改占位符 `{{}}` 中的内容

### 3.3 第三步：调整模型参数

1. 在「模型设置」中调整：
   - Temperature：0.7-0.8
   - Max Tokens：2000-4000
2. 保存设置

### 3.4 第四步：配置技能（可选）

1. 根据需求添加插件
2. 上传相关知识库文档
3. 配置变量（如需要）

### 3.5 第五步：测试与发布

1. 在测试页面与Bot对话
2. 根据反馈微调Prompt
3. 发布到目标渠道

## 四、适配器技术实现

### 4.1 适配器输入格式（Neshama配方）

```json
{
  "soul": {
    "name": "Bot名称",
    "identity": "身份定义",
    "values": ["价值观1", "价值观2"],
    "behaviors": ["行为准则1", "行为准则2"],
    "traits": {
      "trait1": "表现描述",
      "trait2": "表现描述"
    }
  },
  "emotions": {
    "types": ["情绪类型列表"],
    "expressions": "表达方式描述"
  },
  "skills": ["技能列表"],
  "memory": {
    "long_term": "长期记忆要点",
    "strategy": "上下文策略"
  },
  "style": {
    "tone": "语气特点",

  "boundaries": {
```

## 五、已知限制与注意事项

### 5.1 扣子平台限制

| 限制项 | 说明 | 应对方案 |
|-------|------|---------|
| Prompt长度 | 最大20,000字符 | 精简非核心内容 |
| 无人格参数化 | 无法直接设置OCEAN模型 | 在Prompt中用文字描述 |
| 无情绪系统 | 无内置情绪机制 | 在Prompt中模拟情绪表达 |
| 无欲望系统 | 无动机建模 | 在行为准则中体现 |
| 记忆系统受限 | 长期记忆依赖配置 | 详细配置记忆策略 |

### 5.2 映射不完整的部分

| Neshama模块 | 扣子支持度 | 说明 |
|------------|-----------|------|
| 情绪系统 | ⚠️ 部分支持 | 需在Prompt中模拟 |
| 欲望系统 | ❌ 不支持 | 需转化为行为准则 |
| 人格参数化 | ❌ 不支持 | 需用自然语言描述 |
| 成长系统 | ⚠️ 有限支持 | 依赖知识库更新 |

### 5.3 优化建议

1. **Prompt优化**：定期根据对话效果微调Prompt
2. **知识库维护**：定期上传新文档保持知识更新
3. **变量利用**：善用扣子的变量功能模拟记忆
4. **版本管理**：保留Prompt历史版本便于回滚
