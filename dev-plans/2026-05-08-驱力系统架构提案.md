# 驱力系统架构提案 — Drive System Proposal

> 状态：提案阶段 | 版本：v0.1 | 日期：2026-05-08
> 提出者：JOJO | 整理：Seele

## 1. 核心问题

**当前Neshama的NPC是"被动反应"的——有灵魂，但没有欲望。**

现有架构：
```
事件 → 人格(OCEAN) + 情绪 + 记忆 → 行为输出
```

缺失的环节：NPC没有内在驱动力，不会主动追求目标，只能被动响应外部刺激。

**情绪是灵魂的温度计，驱力才是灵魂的发动机。**

## 2. 驱力系统设计

### 2.1 驱力层级（基于马斯洛需求层次，游戏化适配）

| 层级 | 驱力 | 游戏场景示例 | 触发条件 |
|------|------|-------------|---------|
| L1 生存 | Survival | NPC饥饿时寻找食物、遇敌逃跑 | HP/资源低于阈值 |
| L2 安全 | Safety | 守卫主动巡逻、村民加固防御 | 感知到威胁/环境不安全 |
| L3 归属 | Belonging | NPC主动找人聊天、加入阵营 | 社交孤立度过高 |
| L4 尊重 | Esteem | 铁匠追求精湛技艺、战士要证明勇气 | 地位/声望下降 |
| L5 自我实现 | SelfActualization | 贤者探索未知、艺术家创作 | 高开放性+低驱力满足触发 |

### 2.2 驱力与OCEAN人格的映射

驱力不是一刀切的，每个NPC的驱力权重不同：

| OCEAN维度 | 增强的驱力 | 抑制的驱力 |
|-----------|-----------|-----------|
| 高开放性 | 自我实现、归属 | 安全 |
| 高尽责性 | 安全、尊重 | — |
| 高外向性 | 归属、尊重 | — |
| 高宜人性 | 归属 | 尊重（攻击性） |
| 高神经质 | 安全 | 自我实现 |

### 2.3 驱力度量模型

```python
class DriveState:
    """驱力状态 — 每个NPC维护一份"""
    survival: float      # 0.0-1.0，当前满足度
    safety: float
    belonging: float
    esteem: float
    self_actualization: float
    
    # 动态衰减：驱力不是静态的，会随时间自然衰减（未满足→驱力增强）
    decay_rate: Dict[str, float]  # 每种驱力的衰减速率，由OCEAN决定
    
    def dominant_drive(self) -> str:
        """返回当前最强烈的未满足驱力"""
        # 满足度最低 → 驱力最强 → 优先行动
        ...
```

关键特性：
- **动态衰减**：满足度随时间自然下降，越久没满足→驱力越强
- **阈值触发**：满足度低于阈值时，驱力从"后台"进入"前台"，开始影响行为
- **OCEAN加权**：同样的环境，不同人格的NPC关注不同驱力

## 3. 驱力→行为的闭环

### 3.1 当前架构（反应式）

```
外部事件 → EmotionDriver → DecisionMaker → 行为
```

### 3.2 加入驱力后（主动式）

```
驱力衰减 → 驱力未满足阈值？
  ├─ 是 → DriveGenerator生成目标 → DecisionMaker选择行动 → 执行 → 满足度更新 → 情绪反馈
  └─ 否 → 等待外部事件（现有流程）
```

核心变化：
1. **DriveGenerator**：根据最强烈驱力 + 人格 + 记忆，生成可选目标
2. **DecisionMaker扩展**：不仅响应事件，还要评估驱力目标
3. **驱力-情绪反馈环**：驱力长期未满足→情绪偏负面；满足→偏正面

### 3.3 行为生成示例

```
守卫NPC：
  - 安全驱力衰减 → 阈值触发 → DriveGenerator: "巡逻区域确保安全"
  - DecisionMaker选择行动: 巡逻路线A（之前记忆：东门更危险）
  - 执行巡逻 → 未发现威胁 → 安全驱力回升 → 情绪：安心
  
村民NPC（高外向性）：
  - 归属驱力衰减 → 阈值触发 → DriveGenerator: "找人交谈"
  - DecisionMaker选择行动: 去酒馆（记忆：酒馆人多）
  - 找铁匠聊天 → 归属驱力回升 → 情绪：愉悦
  
贤者NPC（高开放性）：
  - 自我实现驱力衰减 → DriveGenerator: "探索未知知识"
  - DecisionMaker: 去图书馆研究古卷
  - 发现新线索 → 记忆更新 → 自我实现驱力回升 → 情绪：兴奋
```

## 4. 与现有系统的集成点

| 现有模块 | 集成方式 | 影响程度 |
|---------|---------|---------|
| OCEANPersonality | 驱力权重来源 | 只读，无修改 |
| EmotionDriver | 驱力→情绪反馈 | 中等，新增驱力影响通道 |
| DecisionMaker | 扩展决策源 | 较大，从纯事件驱动→事件+驱力双驱动 |
| MemorySystem | 驱力行动记忆存储 | 小，新增记忆类型 |
| NPCState | 新增DriveState | 中等，API扩展 |
| BehaviorSystem | 驱力触发的行为类型 | 中等，新增主动行为 |

## 6. 版本规划建议

### V1（当前）— 不纳入
- 专注游戏场景SDK落地
- 情绪+人格+记忆已经够用
- 先让Unity/UE5跑通，收集真实反馈

### V2 — 驱力系统核心
- DriveState + 衰减模型
- DriveGenerator（目标生成）
- DecisionMaker扩展（双驱动）
- 驱力-情绪反馈环

### V3 — 高级驱力
- 驱力冲突（安全vs归属的取舍）
- 群体驱力（一个阵营的共同目标）
- 长期驱力规划（NPC制定并执行多步计划）

## 7. 超出游戏场景的潜力

驱力系统一旦建成，Neshama的应用场景会大幅扩展：

| 场景 | 驱力适配 |
|------|---------|
| AI Agent | 任务驱力（完成目标）+ 学习驱力（获取新知识） |
| 客服系统 | 帮助驱力（解决用户问题）+ 效率驱力（快速响应） |
| 陪伴机器人 | 归属驱力（建立关系）+ 关怀驱力（关注用户状态） |
| 教育NPC | 引导驱力（帮助学生理解）+ 挑战驱力（适度施压） |

但——**先打透游戏，再谈扩展**。
