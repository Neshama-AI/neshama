### R9 — Cpp17已废弃 + BindLambda不兼容DynamicDelegate
- 错误1：UE5.6 `CppStandardVersion.Cpp17` 标记[Obsolete]，UBT无法生成Shared RSP→D8022全部编译失败
- 修复1：→ `CppStandardVersion.Cpp20`
- 错误2：`DECLARE_DYNAMIC_DELEGATE`不支持`BindLambda`，仅支持`BindUFunction`
- 修复2：→ `DECLARE_DELEGATE`（单播委托），`AddLambda`绑定正常
- Commit: 32130c4, 打包v9

### R10 — 非ASCII路径PCH C1083诊断 + Build.cs加固
- 错误：MSVC c1xx不支持非ASCII项目路径→PCH C1083 "No such file"
- 修复：Build.cs加`DefaultBuildSettings = BuildSettingsVersion.Latest;`消除UE5.6升级警告
- Commit: f916e1e, 打包v10

### R11 — 非ASCII路径检测升级为FATAL
- 错误：R10的WARNING级别检测未中止编译，仍走到PCH才报C1083
- 修复：`CheckNonAsciiProjectPath()`从`Console.WriteLine(WARNING)`升级为`throw BuildException(FATAL)`
- 提前中止编译，避免无意义的C1083报错，给用户更清晰的指引
- Commit: 3ef369a, 打包v11

### R12 — SocialTypes.h与引擎同名冲突
- 错误：`Two headers with the same name is not allowed. 'SocialTypes.h' conflicts with Engine/Plugins/Online/OnlineFramework/Source/Party/Public/SocialTypes.h`
- 根因：UHT要求所有反射头文件名全局唯一，我们的`SocialTypes.h`与引擎OnlineFramework插件同名
- 诊断过程：
  1. 日志唯一Error在uhtmanifest(1)行，说明UHT阶段即失败，0个文件生成
  2. 搜索确认`SocialTypes.h`存在于`Engine/Plugins/Online/OnlineFramework/Source/Party/Public/`
  3. 这是UE5内置OnlineFramework的社交系统类型定义，与我们的NPC社交系统类型文件同名
  4. UHT按文件basename去重，不区分目录层级，因此即使路径不同也会冲突
- 修复：
  1. `SocialTypes.h` → `NeshamaSocialTypes.h`（加Neshama前缀消除冲突）
  2. `SocialTypes.cpp` → `NeshamaSocialTypes.cpp`
  3. 更新3处include引用（generated.h、cpp、SocialEngine.h）
- 涉及文件：3个文件修改/重命名
- 类型名检查：`ESocialInteractionType`和`ERelationshipCategory`与引擎不冲突，无需改名
- 主动扫描：检查所有SDK头文件名是否还有潜在冲突（SocialEngine.h/EntityGraph.h/MemorySystem.h等），均为LOW风险
- Commit: 6890e60, 打包v12

### R13 — FailedDueToEngineChange（环境问题，非代码错误）
- 错误：UE5编辑器编译添加`-NoEngineChanges`，阻止引擎模块重建，导致`FailedDueToEngineChange`
- UHT已通过（R12修复生效），159头文件生成成功

- 修复：用户需从Visual Studio编译，或删除Intermediate/Binaries后重建
- 无代码修改
