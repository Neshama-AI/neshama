# 贡献指南

感谢您对 Neshama 项目的兴趣！我们欢迎各种形式的贡献。

---

## 如何贡献

### 1. 报告问题

- 使用 GitHub Issues 提交 Bug 报告
- 描述问题细节：复现步骤、环境信息
- 附上相关截图或日志

### 2. 提出建议

- 通过 GitHub Issues 提出新功能建议
- 描述功能需求和使用场景
- 解释为什么这个功能对社区有价值

### 3. 完善文档

- 修复拼写错误或语法问题
- 补充缺失的文档内容
- 翻译文档到其他语言
- 添加更多示例

### 4. 提交代码

1. Fork 本仓库
2. 创建特性分支：`git checkout -b feature/your-feature`
3. 编写代码并测试
4. 提交：`git commit -m 'Add some feature'`
5. 推送到分支：`git push origin feature/your-feature`
6. 创建 Pull Request

### 5. 编写示例

- 添加新的应用场景示例
- 提供不同平台的适配案例
- 分享最佳实践

---

## 代码规范

- 使用清晰、有意义的变量和函数命名
- 添加必要的注释
- 保持代码风格一致
- 确保代码可通过基本测试

---

## 文档规范

- 使用 Markdown 格式
- 保持语言简洁明了
- 添加必要的示例代码
- 相关文件之间保持链接

---

## 社区准则

- 尊重他人，友善交流
- 对新人保持耐心
- 欢迎不同背景和观点的贡献者
- 保持建设性的讨论氛围

---
